Sorry for the Hiatus & Behind the Scenes of my Active Page JS

I've been stuck on this active page thing for a while. You'd notice if you could see the GitHub Repo of my website. I'll show each of the little snippets so you don't have to dig through it.

First, we start off with the function I made to handle this. It's not very efficient, but nontheless I pulled it off:

  function activePage() {
    if (window.location.href.indexOf("#1") !== -1) { //If you're on the first page
      pageButtonsArray[0].className = "pageButton inflate active";
      pageButtonsArray[1].className = "pageButton inflate";
      pageButtonsArray[2].className = "pageButton inflate";
    }
    else if (window.location.href.indexOf("#2") !== -1) { //If you're on the second page
      pageButtonsArray[0].className = "pageButton inflate";
      pageButtonsArray[1].className = "pageButton inflate active";
      pageButtonsArray[2].className = "pageButton inflate";
    }
    else if (window.location.href.indexOf("#3") !== -1) { //If you're on the third page
      pageButtonsArray[0].className = "pageButton inflate";
      pageButtonsArray[1].className = "pageButton inflate";
      pageButtonsArray[2].className = "pageButton inflate active";
    }
  }

I commented it in, but basically it just checks which page you're on by looking at the href of the page, and changes the pagination buttons on the bottom accordingly. I use a class called "active" to discern which link is active.

createA.addEventListener("click", () => { //Add a click listener to the anchors
  showPage(i - 1); //Run the showPage function for whatever i is clicked
  window.scrollTo(0,0); //Scroll to the top of the page
  setTimeout(activePage, 50); //Wait 50 milliseconds before running the activePage function (so it reads the proper window.location.href.indexOf)
});

For this one, I added to the eventlistener that is already on the pagination buttons. It waits a tiny bit so when you click the button the page has time to change, and then runs the first function to check which link is active.

I want to take this time to quickly go over something else I added, which is a way to view single blog posts by clicking on them. 

function individualPost() {
  let postsToListen = [] //An empty array for posts to add an eventlistener to
  for (let i = 0; i < numberOfPosts; i++) { //Loop throught all of the posts on the page and...
    postsToListen.push($realBlog[i]); //Push each blog post into the postsToListen array
    let postToDisplay = []; //An empty array for deciding which post to display
    postsToListen[i].addEventListener("click", () => { //When you click on a specific post...
      $realBlog.hide(); //Hide every post
      postToDisplay.push(postsToListen[i]); //Push the clicked post into the postToDisplay array
      $(postToDisplay).show(); //Show what's in the postToDisplay array
      $(".blogPageButtons").hide(); //Hide the page buttons at the bottom
      $(".borderBottom").hide(); //Hide the bottom border of the posts
      $backToBlogs.show(); //Show the "Back to the Blogs" div at the bottom of the page
      backToBlogs.addEventListener("click", () => { //When you click on the "Back to the Blogs" div...
        showPage(0); //Show the first page of blog posts
      });
    });
  }
}

Once I figured these two out I though I was in the clear, but not yet. I did some quick testing and found that when I would use my backToBlogs functionality, and then click the "Back to Blogs" button, there would be two pages highlighted.

backToBlogs.addEventListener("click", () => { //When you click on the "Back to the Blogs" div...
  showPage(0); //Show the first page of blog posts
  pageButtonsArray[0].className = "pageButton inflate active"; //Make the first page the active one, since it redirects you back to it
  if (window.location.href.indexOf("#2") !== -1) { //If you were on the second page previously...
    pageButtonsArray[1].className = "pageButton inflate"; //Reset the 2nd page button's classnames
  }
  else if (window.location.href.indexOf("#3") !== -1) { //If you were on the third page previously...
    pageButtonsArray[2].className = "pageButton inflate"; //Reset the 3rd page button's classnames
  }
});

I added a little bit to the backToBlogs eventlistener, so it checks if you were on the second or third page before you viewed a single blog post, and then resets the classes of the corresponding button.

All in all these two functionalities kept me busy for almost a month. There was a lot of testing and hair-pulling, but I got them to work how I intended them to.

A simple mistake

So I've had a problem with my blog section on my personal website for a while... But I found the solution, and I actually facepalmed when the posts started loading. Here's the link to my question for anyone interested.

For anyone who's too lazy, I'll give you a TL;DR. I have this little function here:

function showPage(page) {
          $realBlog.hide();
            let postsToDisplay = [];
            for (let i = 0; i < $realBlog.length; i += 1) {
              if (i >= page * postsPerPage && i <= page * postsPerPage + postsPerPage - 1) {
                postsToDisplay.push($realBlog[i]);
                $(postsToDisplay[i]).show();
                }
              }
              return postsToDisplay;
            }

Now, what it's supposed to do is loop through the blog posts according to the page you're on (the maximum posts per page is 10). So it makes an empty array and pushes the right posts into it, and then shows it. The problem is this little guy right here:

$(postsToDisplay[i]).show();

What I wanted it to do is show each post in the array. The problem is: why am I trying to show each object in the array, when I can just show the whole array? I don't know. The fix is this easy:

$(postsToDisplay).show();

It's always the simplest mistakes that cause the biggest issues, isn't it? I should know that by now.

Anyway, I'm really satisfied now, and everything is working according to plan. I'm gonna do a final run-through of my site, and eventually migrate all of my posts here to my personal site. I'll add one final post here with a link to my personal site once I've finished migrating.
